# Modern Intro/Demo Framework: C++ Architecture

**Building a modular, reusable intro/demo system with lessons from HowIMetYourMod**

This guide describes a fresh C++ architecture using modular libraries, eliminating the Python/C++ split and creating reusable components for future projects.

---

## Philosophy: Learn and Improve

### What We Learned from HowIMetYourMod

**✅ What Worked:**
- Scene-based timeline authoring
- Curve-driven shader animation (0.0-centered design)
- Split shader files (one per scene)
- JSON project format with separate runtime export
- "Do It All" workflow (save → export → build → run)
- Shader presets and randomization

**❌ What to Improve:**
- Python/C++ boundary → **All C++ now**
- tkinter performance → **ImGui at 60 FPS**
- Scattered subsystems → **Modular libraries**
- Large static globals → **Clean ownership**
- Monolithic codebase → **Reusable components**

### New Architecture Goals

1. **Single Language** - C++ for runtime AND editor
2. **Modular Libraries** - Independent, reusable components
3. **Clean APIs** - Explicit contracts, no hidden dependencies
4. **Size Discipline** - Track footprint per library
5. **Modern C++** - C++17/20, best practices

## Library Architecture

### The Seven Core Libraries

```
revision_libs/
├── rev_platform   → Win32 window + WGL + timing + input [~15 KB]
├── rev_shader     → GLSL compilation + uniforms [~10 KB]
├── rev_xm         → XM music playback (libxm wrapper) [~30 KB]
├── rev_curve      → Animation curves with easing [~8 KB]
├── rev_sequence   → Timeline + cue system [~12 KB]
├── rev_mesh       → 3D mesh loading + rendering [~20 KB]
└── rev_editor     → ImGui-based scene authoring [~150 KB]
```

**Minimum Runtime**: rev_platform + rev_shader + rev_xm = ~55 KB  
**Full Runtime**: All runtime libs = ~95 KB  
**Editor**: All libs + ImGui = ~245 KB

### Design Principles

**1. Zero Interdependencies**
- Libraries don't depend on each other
- Runtime links only what it needs
- Editor can use all libraries for preview

**2. Explicit APIs**
- C++ namespaces (`rev::platform`, `rev::shader`, etc.)
- Create/Destroy lifecycle (no RAII complexity)
- Context passed explicitly (no hidden globals)

**3. Header + Source**
- Clean public API in `include/rev_<name>.h`
- Implementation in `src/*.cpp`
- CMake generates static libs

**See [LIBRARY_DESIGN.md](LIBRARY_DESIGN.md) for complete API reference**

## Quick Start: Minimal Intro

### 1. Create Libraries

**Directory structure**:
```
my_intro/
├── libs/                  # Git submodule: revision_libs
│   ├── rev_platform/
│   ├── rev_shader/
│   └── rev_xm/
├── src/
│   └── main.cpp
├── assets/
│   ├── fragment.glsl
│   └── music.xm
└── CMakeLists.txt
```

### 2. Main Runtime

**src/main.cpp**:
```cpp
#include "rev_platform.h"
#include "rev_shader.h"
#include "rev_xm.h"

// Embedded assets (generated by CMake)
extern const char fragment_shader_src[];
extern const unsigned char music_xm_data[];
extern const size_t music_xm_size;

int main() {
    // Platform
    auto* window = rev::platform::CreateWindow(1920, 1080, true, "Intro");
    rev::platform::LoadGLFunctions();
    
    // Shader
    const char* vertex = R"(#version 330
        void main() {
            vec2 p = vec2((gl_VertexID & 1) * 2 - 1, (gl_VertexID & 2) - 1);
            gl_Position = vec4(p, 0.0, 1.0);
        })";
    
    auto* shader = rev::shader::CompileFromSource(vertex, fragment_shader_src);
    int u_time = rev::shader::GetUniformLocation(shader, "u_time");
    int u_resolution = rev::shader::GetUniformLocation(shader, "u_resolution");
    
    // Music
    auto* music = rev::xm::CreatePlayer(music_xm_data, music_xm_size);
    
    // Main loop
    double start = rev::platform::GetTime();
    while (!window->should_close && rev::platform::PollEvents(window)) {
        float time = (float)(rev::platform::GetTime() - start);
        
        // Render shader
        glClear(GL_COLOR_BUFFER_BIT);
        rev::shader::Use(shader);
        rev::shader::SetFloat(shader, u_time, time);
        rev::shader::SetVec2(shader, u_resolution, 1920.0f, 1080.0f);
        glDrawArrays(GL_TRIANGLE_STRIP, 0, 4);
        
        rev::platform::SwapBuffers(window);
        
        if (rev::platform::IsKeyPressed(window, VK_ESCAPE)) break;
    }
    
    // Cleanup
    rev::xm::DestroyPlayer(music);
    rev::shader::DestroyProgram(shader);
    rev::platform::DestroyWindow(window);
    
    return 0;
}
```

**Result**: ~80 KB executable with shader + music

### 3. CMakeLists.txt

```cmake
cmake_minimum_required(VERSION 3.20)
project(MyIntro)

set(CMAKE_CXX_STANDARD 17)

# Add revision libraries
add_subdirectory(libs)

# Embed assets
file(READ "assets/fragment.glsl" FRAGMENT_SRC)
file(READ "assets/music.xm" HEX MUSIC_XM_HEX)
configure_file(src/embedded_assets.h.in embedded_assets.h @ONLY)

# Runtime executable
add_executable(intro src/main.cpp)
target_link_libraries(intro PRIVATE rev_platform rev_shader rev_xm)
target_include_directories(intro PRIVATE ${CMAKE_BINARY_DIR})
```

## Editor Application

### Why C++ Editor vs. Python?

**Advantages**:
1. **Same language** - No Python install, no subprocess calls
2. **Performance** - ImGui renders at 60+ FPS, even with large timelines
3. **Live preview** - Render intro inside editor viewport
4. **Better debugging** - Single process, unified tooling
5. **Smaller distribution** - 250 KB vs. 50+ MB (Python + tkinter)

### Editor Structure

**editor_main.cpp**:
```cpp
#include "rev_platform.h"
#include "rev_editor.h"

int main() {
    // Windowed mode for editor
    auto* window = rev::platform::CreateWindow(1600, 900, false, "Scene Editor");
    rev::platform::LoadGLFunctions();
    
    // Editor context
    auto* editor = rev::editor::CreateEditor(window);
    rev::editor::LoadProject(editor, "project.json");
    
    // Main loop
    while (!window->should_close && rev::platform::PollEvents(window)) {
        glClear(GL_COLOR_BUFFER_BIT);
        
        rev::editor::BeginFrame(editor);
        rev::editor::RenderUI(editor);
        rev::editor::EndFrame(editor);
        
        rev::platform::SwapBuffers(window);
    }
    
    rev::editor::DestroyEditor(editor);
    rev::platform::DestroyWindow(window);
    
    return 0;
}
```

### Editor Features

**Timeline View** (DAW-style):
- Horizontal timeline with scene blocks
- Drag to reorder, resize to adjust duration
- Add/delete scenes, effects, cues
- Zoom and pan

**Shader Modal**:
- Shader preset dropdown
- Color pickers (palette_low, palette_mid, palette_high)
- Parameter sliders (speed, intensity, warp, exposure, fade)
- Randomize buttons (colors, curves, values)
- Reset to defaults

**Curve Editor**:
- Canvas-based curve drawing
- Add/delete/move points
- Easing controls (in/out tangents)
- Segment modes (linear, ease_in/out, smoothstep, hold)
- Target selection (common, shader-specific)

**Live Preview Viewport**:
- Embedded OpenGL render of intro
- Scrub timeline to any time
- See shader changes immediately

**Export & Build**:
- "Export" button → generate `cues.txt` from project JSON
- "Build" button → run CMake + compile
- "Run" button → launch intro.exe
- "Do It All" button → all three in sequence

## Best Practices

### Code Organization

**Namespaces**:
```cpp
namespace rev::platform { ... }
namespace rev::shader { ... }
namespace rev::xm { ... }
// etc.
```

**Naming Conventions**:
- Functions: `PascalCase`
- Types: `PascalCase`
- Variables: `snake_case`
- Members: `snake_case_` (trailing underscore)
- Constants: `kPascalCase`

**File Structure**:
```
revision_libs/rev_<name>/
├── include/rev_<name>.h   # Public API only
├── src/
│   ├── <feature_1>.cpp
│   ├── <feature_2>.cpp
│   └── <name>_internal.h  # Private helpers
└── CMakeLists.txt
```

---

### Memory Management

**Explicit Ownership**:
```cpp
// Create returns allocated pointer
Window* CreateWindow(...);

// Destroy frees memory
void DestroyWindow(Window* window);

// Caller responsible for lifetime
auto* win = CreateWindow(...);
// ... use window ...
DestroyWindow(win);
```

**No RAII** (for size):
- Manual Create/Destroy keeps code simple
- No C++ destructor overhead
- Explicit cleanup in main()

**Stack vs. Heap**:
- Small data on stack (`float[16]`, `Point`, etc.)
- Large arrays on heap (`std::vector`, manual `new[]`)
- GPU buffers stay on GPU (VBO/IBO, not CPU side)

---

### Size Discipline

**Measure Everything**:
```bash
# Per-library size
size -A librev_platform.a
size -A librev_shader.a

# Final executable
size intro.exe
```

**Avoid Bloat**:
- No STL containers in hot paths (when size matters)
- Prefer C arrays + manual management
- No exceptions (compile with /EHs-c-)
- No RTTI (compile with /GR-)
- Minimal includes (forward declarations)

**Compression**:
- kkrunchy: 3-4x typical
- Crinkler: 4-5x (better for small exes)
- Target <25 KB for 64k intro

## Conclusion

This modern C++ architecture eliminates the Python/C++ split, creating a unified, modular system with:

✅ **Single language** - C++ for everything  
✅ **Reusable libraries** - Drop into any project

The seven core libraries (`rev_platform`, `rev_shader`, `rev_xm`, `rev_curve`, `rev_sequence`, `rev_mesh`, `rev_editor`) provide everything needed to build competition-ready intro/demo productions.
